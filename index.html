96 不同的二叉搜索树

问题描述

给你一个整数 n ，求恰由 n 个节点组成且节点值从 1 到 n 互不相同的 二叉搜索树 有多少种？返回满足题意的二叉搜索树的种数。

示例1：

问题分析

二叉搜索树的定义

二叉搜索树应具备以下几种性质：

  1. 空树
  2. 如果不为空树：如果它的左子树不为空，左子树上的所有节点应该小于根节点的值，如果右子树不为空，右子树上的所有节点都应该大于根节点的值。

问题分析

对于从 $1$ 到 $n$ 的整数序列，求该序列能够组成多少中不同的二叉树应该遍历该序列中的每个数字 $i$，将该数字作为根节点，$$[1, i-1]$$中的数字作为左子树，$$[i+1, n]$$ 中的数字作为右子树， 然后递归的构建。

由此，可以看出，原问题可以分解成左右子树的构建问题，使用动态规划求解。

对于给定序列 $$1$$ ··· $$n$$ , 假设能够构成 $$G(n)$$ 个二叉搜索树，如果选择数字 $i$ 作为根节点，则：

$$1$$·····$$i - 1$$ 能够构建的左子树数量为$$G(i-1)$$, $$i+1$$ ···· $$n$$ 能够构建的右子树数量为$$G(n-i)$$,

因此$$G(n)=\sum_i^n{G(i-1)G(n-i)}$$

动态规划

dp数组及下标含义

$$dp[i]$$表示 $1$ ···· $i$ 能够构成多少个不同的二叉搜索树

递推公式

$$dp[i]+=dp[j-1]\times dp[i-j]$$

初始化

当$$i = 0$$时，$$dp[0] = 1$$

当$$i = 1$$时，$$dp[1] = 1$$;

遍历顺序

对于 $i$ 来说，需要从2开始，遍历到n结束，对于j来说，需要从1开始，到i结束。

整数拆分

1. 问题描述

给定一个正整数 n，将其拆分为至少两个正整数的和，并使这些整数的乘积最大化。 返回你可以获得的最大乘积。

示例 1:

示例 2:

说明: 你可以假设 n 不小于 2 且不大于 58。

2. 题解

2.1 问题分析

对于一个正整数 $$n$$， 当 $$n >= 2$$ 满足拆分至少两个整数和的要求。具体拆分有以下两种情况：

  1. 拆分成为两个数：拆分的第一整数为 $$k$$, 另一个数为 $$n - k$$
  2. 拆分成为两个以上的数：对 $$n - k$$继续拆分

由此可见，正整数 $$n$$ 拆分得到的整数最大乘积取决于比它小的正整数得到的最大乘积，是一个最优子问题，可以使用动态规划。

2.2 动态规划求解五步

确定dp数组以及下标含义

创建dp数组，$$dp[i]$$ 表示将整数 $$i$$ 拆分成为至少两个整数的和之后，对应的这些整数的最大乘积。

确定递推公式

整数 $$i (i\geq\ 2)$$, 拆分得到的第一个整数为 $$j$$ $$(1 \leq\ j < i)$$

拆分策略：

  1. 拆成两个整数的和，即：$$i$$ 与 $$i - j$$， 此时乘积为 $$j \times (i - j)$$
  2. 拆分成两个以上的整数和，此时乘积为: $$j \times dp[i - j]$$

因此当$$i$$, $$j$$ 都固定的时候，此时有：

$$dp[i] = max( j \times (i - j), j \times dp[i -j])$$

又因为 $$j$$ 要从 $$1$$到$$(i - 1)$$ 进行遍历，取所有 $$j$$ 能够达到的最大值，所以此时的递推公式为：

$$dp[i] = max(dp[i], max( j \times (i - j), j \times dp[i -j]))$$

dp数组的初始化

$$dp[0], dp[1]$$ 无任何实际意义，将$$dp[2]$$ 初始化为1即可

确定遍历顺序

由递推公式得， $$dp[i]$$ 的值依赖于 $$dp[i-j]$$ 的值，因此遍历i，是从3开始，到n结束，$$i \in [3, n]$$ , 从小到大遍历

j是从1到i-1

动态规划问题

动态规划，如果某一个问题有很多重叠的子问题，使用动态规划。动态规划中的每个状态都是由上一个状态推导出来的。

动态规划的解题步骤

对于动态涉及到动态规划的问题，首先要分析题目，有的题目还要将问题进行简化以及转化，具体的：

  1. 确定dp数组以及下标的含义
  2. 确定递推公式
  3. dp数组初始化的方式
  4. 确定遍历顺序
  5. 举例推导dp数组

debug的最佳方式就是把dp数组打印出来，与自己推导的结果对比。

第7章 类

7.1 定义抽象数据类型

  1. 类的基本思想：数据抽象（data abstraction）和封装（encapsulation）
  2. 数据抽象：接口与实现分离的编程技术。
    1. 接口是用户所能执行的操作
    2. 实现包括类的数据成员、负责接口实现的函数体以及定义类所需要的各种私有函数

7.1.1 设计Sales_data类

解决书店程序，Sales_data类的作用是表示一本书的总销售额、售出册数以及平均售价。
Sales_data的接口应该包含以下操作：

  1. 一个isbn函数，用于返回对象的ISBN编号
  2. 一个combine的成员函数，用于将一个sales_data对象加到另一个对象上
  3. 一个add函数，执行两个对象的加法（其中，该函数不作为Sales_data的成员函数，具体原因见14.1节 490页）
  4. 一个read函数，将数据从istream读入到Sales_data对象中
  5. 一个print函数，将Sales_data对象的值输出到ostream

Sales_data的成员函数应该包括：

  1. bookNo，书籍的ISBN号
  2. units_sold, 书籍的销量
  3. revenue，书籍的总销售收入

7.1.2 初探C++类

this指针

对象对成员函数机型进行调用时，比如：
total.isbn()
使用点运算符来访问total对象的isbn成员，然后调用。
成员函数实际上是通过一个名为this的隐式参数来访问调用它的那个对象。具体的可以看作成员函数isbn()中
有个隐式形参this指针：isbn(this)当调用一个成员函数的时候，用请求该函数的对象地址初始化this。

调用total.isbn()时，编译器负责把total的地址传送给isbn的隐式形参this。
this是一个常量指针，不允许改变this中保存的地址。

const成员函数

isbn函数在参数列表之后紧随一个const关键字。const的作用是修改隐式this指针的类型

默认情况下，this的类型是指向类类型非常量版本的常量指针。例如：
Sales_data成员函数中，this的类型是Sales_data *const。所以在默认情况下，不能把this绑定到一个常量对象
上，因此，不能在一个常量对象上调用普通的成员函数。
常量成员函数的this是指向常量的指针，所以常量成员函数不能改变调用它对象的内容。如果成员函数不改变对象的状态，最好将
成员函数声明成为常量成员函数，以提高函数的灵活性。

常量对象以及常量对象的引用或指针只能调用常量成员函数

7.1.3 定义类相关的非成员函数

类的作者通常需要定义一些辅助函数，这些函数从概念上属于类的接口部分，但是实际上不属于类本身。这样的函数应该与类声明定义
在同一个头文件中。

7.1.4 构造函数

构造函数：类通过一个或几个特殊的成员函数来控制其对象的初始化过程，

合成的默认构造函数（synthesized default constructor）：编译器创建的构造函数，初始化规则：

  1. 存在类内初始值，使用初始值初始化，否则默认初始化

注意：

  1. 只有当类没有声明任何构造函数的时候，编译器才会自动生成默认构造函数
  2. 某些类不能依赖与合成的默认构造函数

手动声明内置构造函数

类似的，在参数列表后加上=default可以要求编译器生成默认构造函数
Sales_data() = default;

7.2 访问控制与封装

上一节中使用的struct组织Sales_data类，不能实现封装功能，类的所有实现均暴露在用户下，C++中可以
使用访问说明符加强类的封装性。

  1. 定义在public关键字之后的成员在整个程序内可以被访问，定义类的接口
  2. 定义在private之后的成员可以被类的成员函数访问，但是不能被使用该类的代码访问，private实现封装
  3. 重写Sales_data类，使用class关键字代替struct

class与struct关键字

class与struct都可以定义一个类，但是他们的默认访问权限不一样。struct定义的类访问权限默认是public
的，class成员的默认访问权限是private的。
当我们希望定义的类的所有成员是public时，使用struct，反之，如果希望成员是private的，使用class

7.2.1 友元

令其他类或者函数成为它的友元，就可以访问类的非公有成员。在函数名或者类名前加上friend关键字即可声明。
一般来说，最好在类定义开始或结束前的位置集中声明友元。

友元的声明

友元的声明仅仅指定了访问权限，而非一个通常意义上的函数声明。如果需要调用友元函数，需要在类的友元声明之外
再专门对函数进行一次声明。
为了使友元对类的用户可见，通常将友元的声明与类本身放在同一个头文件中。
